# 🎒 The Data Analyst's Survival Kit

Willkommen beim großen Git-Abschlussprojekt! 🎉 

Dieses Repository ist unser gemeinsames "Überlebenshandbuch" für den IT-Alltag. Wenn wir fertig sind, enthält es geniale Python-Tricks, Lebensretter-SQL-Queries, tiefgründige Tech-Haikus und Bilder von Haustieren, die uns beim Debuggen helfen. 

Am Ende dient dir dieses Projekt als **echtes Portfolio** für dein GitHub-Profil. Du beweist damit, dass du den Git-Workflow im Team beherrschst.

---

## 🎯 Deine Mission

Bitte lies dir diese Anleitung **Schritt für Schritt** durch. Du arbeitest hier genau so, wie Entwickler in echten Open-Source-Projekten arbeiten.

### Schritt 1: Issue sichern
1. Gehe oben auf den Reiter **Issues**.
2. Such dir eine Aufgabe aus, die dir gefällt, um ins Repository beizutragen.
3. Hinterlasse gerne auch ein Kommentar unter deinem ausgewählten Issue. 

### Schritt 2: Forken, Klonen & die Zeitreise ⏱️
1. **WICHTIG:** Damit wir alle vom selben Startpunkt loslegen, erstellen wir unseren Arbeits-Branch ausgehend von einem eingefrorenen Template-Branch (NICHT von `main`).
2. Klicke oben rechts auf **Fork**, um eine Kopie in deinem Account zu erstellen. **Achte hier unbedingt darauf**, dass das Häkchen bei *"Copy the `main` branch only"* **deaktiviert** ist, damit alle Branches geforkt werden!
3. Klone deinen Fork nun lokal über eine GUI oder das Terminal.
4. Erstelle ausgehend vom Branch `base-template` einen **neuen eigenen Branch** (z. B. `feature-dein-name`) und wechsle dorthin. Jetzt können wir arbeiten!

### Schritt 3: Die Aufgabe lösen
Erledige die Aufgabe aus deinem Issue.
* Code-Snippets (`.py`, `.sql`, `.md`) speicherst du im Ordner `/snippets`.
* Bilder speicherst du im Ordner `/assets`.
* Du kannst gerne auch neue Unterordner erstellen.

### Schritt 4: Die "Hall of Fame" (ACHTUNG: Hier lauert die Falle! 🪤)
Damit dein Beitrag gültig ist, musst du dich in unsere `CONTRIBUTORS.md` eintragen. 
**WICHTIGE REGEL:** Trage deinen Namen (oder einen Spitznamen) **zwingend in Zeile 3** anstelle des Kommentars ein! 

### Schritt 5: Commit & Push
Speichere deine Änderungen (Commit mit einer aussagekräftigen Nachricht) und pushe diese auf *deinen* Fork bei GitHub. Dein neuer Branch sollte nun online auf deinem Profil sichtbar sein.

### Schritt 6: Der Pull Request (PR)
Gehe auf GitHub in *deinen* Fork. Klicke auf den grünen Button **"Compare & pull request"**. Verlinke in der Beschreibung dein Issue (z. B. "Closes #5") und erstelle den PR gegen unser Original-Repository.

---

## ⚔️ Der Endgegner: Der Merge-Konflikt

Da **jeder** von euch seinen Namen exakt in Zeile 3 der `CONTRIBUTORS.md` einträgt, wird GitHub meckern: **"This branch has conflicts that must be resolved."**

Das ist kein Fehler – das ist pure Absicht! Willkommen in der Realität der IT. 😉

**Deine finale Aufgabe:**
Löse diesen Konflikt! Du darfst deinen Namen nicht löschen, aber die der anderen auch nicht. 
* Klicke in deinem PR auf **"Resolve conflicts"**.
* Entferne die Git-Marker (`<<<<<<<`, `=======`, `>>>>>>>`).
* Ordne deinen Namen einfach in der Liste unter oder über den bereits vorhandenen Namen ein.
* Speichere und schließe den Merge ab.

Sobald der PR konfliktfrei ist (grünes Häkchen), wird dein Code ins Survival Kit aufgenommen! 🚀
